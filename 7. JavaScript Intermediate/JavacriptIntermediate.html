Task #3

In your source code you've noticed that you have a function 


					function process(a, b, c, d, e) { 
						return 10000*a+1000*b+100*c+10*d+e; 
					}

which is called 10 times where a,b,c,d params are the same and only e param differs.
Create a universal schonfinkelize(...) function, which will receive any function as a first param, and a list of args which are the same during multiple function calls and will return a function with only one param which is used.

(Search for currying/schonfinkelisation)

Sample: 


					function process(a, b, c, d, e) { 
						return 10000*a+1000*b+100*c+10*d+e; 
					}

In our sample: 


					var newProcess = schonfinkelize(process, 1, 2, 3, 4);
					newProcess(5); // 12345

					Other possible usages:
					schonfinkelize(process, 1)(2, 3, 4, 5);  // 12345
					schonfinkelize(process, 1, 2)(3, 4, 5);  // 12345
					schonfinkelize(process, 1, 2, 3)(4, 5);  // 12345
					schonfinkelize(process, 1, 2, 3, 4)(5);  // 12345

Task #4

Create a widget that will provide next 3 loggers:

  1. Function which will create actionLogger object. That object will contain methods such as:
    • showLog() - displays in cosole all actions being logged
    • addRecord(record) - adds specific text log record added by user code
    • clear log() - clears logs
  2. Function which will create mouseEventLogger object. That object will contain methods such as:
    • showLog() - displays in cosole all actions being logged
    • addRecord(record) - adds specific text log record added by user code
    • clear log() - clears logs
    • logMouseClick(clickEvent) - method will be triggered in click event handlers and contain mouse pointer coords and other click specific info
  3. Function which will create keyboardEventLogger object. That object will contain methods such as:
    • showLog() - displays in cosole all actions being logged
    • addRecord(record) - adds specific text log record added by user code
    • clear log() - clears logs
    • logButtonPressed(buttonPressedEvent) - method will be triggered after keyboard button is pressed and will contain all specific info about the event.

Think about:

  • "Private" and "public" methods
  • Common functionality
  • oop principles
