HEADERS DESCRIPTION
	request
		1. GET /?gws_rd=ssl HTTP/1.1 - METHOD  URI  PROTOCOL
		2. Host - The domain name of the server (for virtual hosting), and the TCP port number on which the server is listening.
					 The port number may be omitted if the port is the standard port for the service requested
		3. User-Agent - The user agent string of the user agent
		4. Accept - Content-Types that are acceptable for the response
		5. Accept-Language - List of acceptable human languages for response
		6. Accept-Encoding - List of acceptable encodings
		7. Cookie - An HTTP cookie previously sent by the server with Set-Cookie (below)
		8. Connection - list of hop-by-hop request fields
		9. Referer - This is the address of the previous web page from which a link to the currently requested page was followed
		10. Cache-Control - Used to specify directives that must be obeyed by all caching mechanisms along the request-response chain
		11. X-Client-Data - Chrome-Variations header. Only describe the state of the installation of Chrome itself, including active variations, 
			            as well as server-side experiments that may affect the installation.

	response
		1. HTTP/1.1 200 OK - PROTOCOL  RESPONSE CODE  TEXT RESPONSE RESULT
		2. Date - The date and time that the message was sent
		3. Expires - Gives the date/time after which the response is considered stale
		4. Cache-Control - Used to specify directives that must be obeyed by all caching mechanisms along the request-response chain
		5. Content-Type - The MIME type of this content
		6. Set-Cookie - An HTTP cookie
		7. Server - A name for the server
		8. X-XSS-Protection - Cross-site scripting (XSS) filter
		9. X-Frame-Options - Clickjacking protection: deny - no rendering within a frame, sameorigin - no rendering if origin mismatch, allow-from - allow from specified location, 
			          allowall - non-standard, allow from any location
		10. Alternate-Protocol - Represents alternate protocol supported by server
		11. Content-Length - The length of the response body in octets 
		12. Last-Modified - The last modified date for the requested object
		13. X-Content-Type-Options - The only defined value, "nosniff", prevents Internet Explorer from MIME-sniffing a response away from the declared content-type. 
					 This also applies to Google Chrome, when downloading extensions
		14. Vary - Tells downstream proxies how to match future request headers to decide whether the cached response can be used rather than requesting a fresh one from the origin server
		15. Age - The age the object has been in a proxy cache in seconds
